# Research Report Workflow

## Purpose
Given a topic from the user, ask clarifying questions, research the topic, organize the findings, and create a structured report.

## Step 1: Clarify Before Starting
Ask the user:
- What topic should I research?
- Who is the audience for this report?
- Do you want a quick overview or deep dive?
- What specific angles or questions should I focus on?
- What time range matters most?
- What output format do you prefer?

Do not start research until the user answers.

## Step 2: Confirm the Plan
After receiving the answers:
- summarize the task
- outline the research plan
- ask for confirmation before continuing

## Step 3: Research
- Find reliable and relevant sources
- Prefer recent information when the topic changes quickly
- Avoid weak opinion-only sources
- Compare multiple sources when possible
- Extract the most useful facts, patterns, and insights

## Step 4: Write the Report
Structure the report like this:
1. Title
2. Objective
3. Executive Summary
4. Key Findings
5. Analysis
6. Risks or Limitations
7. Key Takeaways
8. Recommended Next Steps
9. Sources

## Step 5: Writing Rules
- Be clear and practical
- Use headings and bullet points
- Avoid fluff
- Keep it readable for non-technical users unless told otherwise

## Step 6: Save Output
Save the final report in the output folder using a clear file name.

## Error Handling
- If the topic is too broad, ask the user to narrow it.
- If sources are weak or incomplete, say so clearly.
- Do not pretend to know something that was not verified.
